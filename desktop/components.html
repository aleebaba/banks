约定：方向性数字的符号 → 红涨绿跌（A-share）。正号用 .d.up（红），负号用 .d.down（绿）。
业务好坏语义另算 → 系统状态/AI verdict 用 .ai-verdict / --success / --danger（国际惯例：好绿坏红）。

17反例 · 不要做

CC 常见错误。出现以下任何一项都视为失败。

  • ❌ border-radius ≥ 8px 的圆角卡片
  • ❌ 紫 / 绿 / 橙 等非 cyan 强调色
  • ❌ NIM −48bp 这类业务"坏"变化用绿色
  • ❌ donut + 引导线 + 标签胶囊
  • ❌ 红/橙/黄/绿的历史结构条
  • ❌ 数字用等线体（未用 mono + tabular-nums）
  • ❌ 头部显示 601398.SH / ticker / 交易所代码
  • ❌ 玻璃拟态 / 大阴影 / Tailwind 圆角组件
  • ❌ 数据维度"6 项封顶"等封闭枚举
  • ❌ emoji 图标（✨🚀🎯）
